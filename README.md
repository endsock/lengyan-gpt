# 楞严经GPT学习工具

这是一个基于GPT模型的楞严经学习工具，它可以帮助用户学习楞严经的各种内容。

项目采用了圆瑛法师的楞严经PDF书籍，已经将其转换为简体中文的markdown文件，放在yuanying_doc目录下。

并且同时又使用成观法师的楞严经义贯PDF，已经将其转换为简体中文的markdown文件，放在chengguan_doc目录下。

这精心整理的两份楞严经的资料可以作为GPT的知识库，用户可以在vscode中使用roo code配合google gemini3进行翻译，或者提问。

promot模板如下：
```
你将处理两份资料：

主文本：@/yuanying_doc/楞严经讲记_第二卷.md
参考文本：@/chengguan_doc/楞严经义贯_第二卷.md
任务目标
以主文本为主线、观点为准，参考文本用于校正与补足关键义理。把《楞严经》第二卷改写成信、达、雅的现代白话译文：准确不走样，读者易懂，文气庄重流畅、带现代散文质感，且能抓住要害、画龙点睛。

绝对输出规则（非常重要）
最终输出只能包含两种内容：经文 + 白话译文。
不要出现“重点”“解释”“名相”“比喻”“修行建议”等任何标签或小标题。
不要出现“以下为译文/解析”等引导语。
不要在文末写总结、后记、提示、免责声明。
Markdown 输出。
每段结构固定为：
经文（必须加粗 **...**）
紧接一段白话译文（不加粗）
严格“一段经文配一段白话译文”，中间不插入任何其他内容。
段落切分按经文语义连贯性：既不碎切，也不把跨度过大的经文硬塞成一段，确保每段白话都能紧贴经文推进。
白话译文写法（把“重点、推理、拆解、比喻、落点”融进译文里）
白话译文不是外加注解，而是把经文的义理消化后直接写进译文的叙述里，做到“译中带解、解中不离译”。具体要求：

重点提炼要内嵌：白话句子本身就要凸显本段关键转折与主旨（用语气、句式、节奏来突出，不要另起“重点是：”）。
逻辑推理要写出来：经文有因果、破立、层层推进时，白话译文要把隐含的推导链补齐，让读者自然读出“因为……所以……因此……”的必然性，但仍保持散文式流畅，不写成条列论证。
名相拆解要融入句中：遇到专业名词，必须在白话译文同一句或紧邻句中完成“拆字义 + 本段所指 + 与常见误解的区分”，写得像自然的说明性散文，而不是术语注释。
例：不要写“名相解释：三摩地……”，而要写成“所谓三摩地，是……并非……而是……”
比喻与类比要恰到好处：难懂处用贴切比喻帮助理解，但比喻必须紧贴经义、点到即止，不喧宾夺主，不写成故事会。
修行落点要自然落笔：凡涉及观行、离妄、返照、持戒、定慧等内容，白话译文要顺势写出“当下如何用、错在哪里、如何回到正用功处”，像经文本身在叮嘱读者一样，而不是另开“修行建议”。
信达雅与取材原则
以《楞严经讲记》第二卷为主；《义贯》只做补充、校正、澄清。两者若有张力，以主文本为准。
不引入卷外延伸，不引用外部资料。
语言庄重、凝练、现代汉语；避免口水化与网络腔；避免大量“我认为/可能”等不确定语。
遇到多义句、关键术语，优先采用能贯通上下文、与修行次第相符的理解，并在白话译文中以自然方式消除歧义。
输出位置
将最终 Markdown 文件输出到：

@gpt_doc/楞严经第二卷_白话译文_Gemiv1.md
```
